    When \texttt{dtype=int32} is left out, the datalist
assumes the default return type of \texttt{loadtxt} (floating point).
Since floating point numbers are stored with limited precision in
Python, it is forced to approximate the integer as a float. As a result,
the last few digits of the integer may be off. Since the values of two
neighboring data points we are subtracting to obtain \texttt{dt} usually
lie very close since they occur right after another, therefore errors
introduced by the floating point approximation results in a visible
difference when we are trying to compare its difference with a
subsequent datapoint as shown in the plot below.

    \subsection{Standard deviation of the mean}
    Looping through \texttt{nstep} so that we can change the size of the
dataset chunk and compute the mean for each. Then use the mean that we
found as the average so that values in \texttt{marr} has something to
compare against:
\[ s=\sqrt{\frac{\displaystyle  \sum^N_{i=1}(\mu_i-\mu)^2}{N-1}}\]
